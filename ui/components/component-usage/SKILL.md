# Component Usage Documentation

Use this skill to document how the component fits into user-facing workflows and UI composition.

## Purpose

Explain concisely:

* Primary responsibility
* Business purpose
* User-facing purpose
* Why the component exists

## When To Use

Describe:

* Common use cases
* Typical workflows
* Screens or pages where it belongs

## When NOT To Use

Describe:

* Anti-patterns
* Common mistakes
* Alternative components

## Visual Description

Describe appearance and behavior without assuming screenshots are available:

* Layout
* Appearance
* States
* Responsive behavior
* Animations
* Interactions

## Accessibility

Document:

* Keyboard support
* ARIA attributes
* Screen reader behavior
* Focus management
* Accessibility concerns
* Accessibility requirements

Call out missing accessibility behavior if source or tests show gaps.

## Usage Examples

Provide:

* Minimal Example
* Typical Example
* Advanced Example
* Real Project Example

Use real examples from the codebase whenever possible. If no real usage exists, provide plausible generated examples and label them as generated.

## Composition Patterns

Describe:

* Parent components
* Child components
* Recommended combinations
* Common layouts

Explain how the component fits into larger workflows.

## Styling Rules

Document:

* CSS classes
* Tailwind conventions
* Theme integration
* Design tokens
* Layout constraints

Include styling restrictions coding agents must follow.
